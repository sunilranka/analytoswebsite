Classifier

Analytos Classifier toolkit supports a number of models for classification, with a clean and consistent programming interface.

 

Why Use a classifier?

Classification is the problem of making a discrete prediction using training data. The key difference between regression and classification is that in regression the target is continuous while in classification, the target is categorical. A classifier can be used for several applications including:

  • Fraud Detection
  • Outlier Detection
  • Personalization
  • Click-through rate prediction
  • Churn Prediction
  • Object recognition from Images
  • ...and many more uses.

The Analytos Difference  :  Classifier Toolkit

There are several challenges in creating a flexible, accurate, and scalable classification toolkit. Open source packages typically contain implementations that are missing either flexibility, accuracy, or scalability. The classifier toolkit contains state of the art models for binary and multiclass classification, which can work with terabytes of data. We elaborate on the 4 major differentiating factors of our classifier toolkit: Comprehensive collection of models

 

Classifier Toolkit supports a range of classification models, all of which provide interfaces to create the model and predict outputs:

  • Logistic regression
  • Nearest neighbor classifier
  • Support vector machines (SVM)
  • Boosted Decision Trees
  • Random Forests
  • Neural network classifier (deep learning)

All of our models use intelligent defaults that ensure they work out of the box without the need for parameter tuning.

 

Unified API

In addition to the basic models, the recommender toolkit offers the following options:

Even though the classifier toolkit contains several algorithms that may differ in how they make predictions, they conform to the same interface which makes it easy to switch out models for those that work better in that application. With all models, call create() to create a model, classify() to make predictions on the returned model, and evaluate() to measure performance of the predictions.

 

Works with all your data

All models can incorporate the following set of rich feature types:

  • Numeric features
  • Categorical features
  • Sparse features (i.e feature sets that have a large set of features, of which only a small subset of values are non-zero)
  • Dense features (i.e feature sets with a large number of numeric features)
  • Text data
  • Images

In addition to these feature types, models like the neural networks, boosted decision trees, and random forests can be used for feature extraction. Using feature extraction, you can “extract” the important features from your dataset using a trained model. These features can then be used as inputs to other models in other intelligent applications.

 

Model Selection Smarts!

It isn't always clear that we know exactly which model is suitable for a given task. The model selector automatically picks the right model for you based on statistics collected from the data set. This is especially useful when you aren’t sure which model is going to work best for your application.
